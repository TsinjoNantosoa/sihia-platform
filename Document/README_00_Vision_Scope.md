# Vision et scope du projet

## Objectif principal
Construire un systeme intelligent de gestion hospitaliere en mode SaaS, d abord en MVP, puis en extension IA avancee.

## Scope MVP (obligatoire)
- Gestion patients
- Gestion medecins
- Gestion rendez-vous
- Auth JWT + RBAC
- Dashboard KPI basique
- Prediction Prophet 7 jours
- Deploiement local Docker Compose

## Hors scope MVP (reporte)
- LSTM 30 jours en production
- Chatbot medical complet multilingue
- HL7 FHIR complet
- Mobile native React Native
- Marketplace modules

## KPIs de succes MVP
- Temps API p95 < 300 ms
- Uptime local/test > 99%
- 0 faille critique detectee sur audit de base
- 2 a 3 cliniques pilotes capables d utiliser la demo

## Risques majeurs
- Scope trop large
- Mauvaise qualite de donnees
- Complexite reglementaire
- Delais de vente B2B sante

## Mitigation simple
- Travailler en sprints de 1 semaine
- Faire une demo executable chaque fin de sprint
- Reporter les options complexes au post-MVP
